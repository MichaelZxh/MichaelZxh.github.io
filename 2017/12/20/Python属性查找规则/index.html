Python属性查找规则

发表于 2017-12-20   |   分类于 Python   |   热度 ℃

参考链接：http://www.cnblogs.com/xybaby/p/6270551.html

如果obj是某个类的实例，那么obj.name(以及等价的getattr(obj,’name’))首先调用__getattribute__。
如果类定义了__getattr__方法，那么在__getattribute__中抛出AttributeError的时候就会调用到__getattr__，而对于描述符(__get__)的调用，则是发生在__getattribute__内部的。

实例属性查找规则

obj = Clz()，那么obj.attr顺序如下：

  1. 如果attr是出现在Clz或其基类的__dict__中，且attr是数据描述器，那么调用其__get__方法；
  2. 如果attr出现在obj的__dict__中，那么直接返回obj.__dict__['attr'];
  3. 如果attr出现在Clz或其基类的__dict__中：
    • 如果attr是非数据描述器，那么调用其__get__方法；
    • 返回__dict__['attr']
  4. 如果Clz有__getattr__方法，调用__getattr__方法；
  5. 抛出AttributeError

属性查找：
第一优先级：类中定义的资料描述器
第二优先级：实例的__dict__属性
第三优先级：类中的__dict__属性，该优先级中非资料描述器优先于__dict__属性
第四优先级：类的__getattr__方法

类属性查找规则

类也是对象，类是元类(metaclass)的实例，所以类属性的查找顺序基本同上。区别在于第2步，由于Clz可能有基类，所以实在Clz及其基类的__dict__中查找attr，注意这里的查找并不是直接返回clz.__dict__['attr']，具体来说，第二步分为以下两种情况：

  • 如果clz.__dict__['attr']是一个descriptor（不管是否资料描述器），都调用其__get__方法
  • 否则返回clz.__dict__['attr']

属性赋值

Python的属性赋值也会受到描述器的影响，同时也会受到__setattr__函数的影响。
对于属性赋值，obj = Clz()，那么obj.attr = var，按照如下顺序：

  1. 如果Clz定义了__setattr__方法，那么调用该方法；
  2. 如果attr出现在Clz或其基类的__dict__中，且attr是资料描述器，那么调用其__set__方法
  3. 等价调用obj.__dict__['attr'] = var
